IMC Virtual Staining — Session Summary
=======================================
Date: 2026-06-20

========================================
EXPERIMENTAL CONDITIONS
========================================

1. Per-marker ResNet-UNet (PCA)
   Preprocessing: SGDRegressor per input channel regressing from 3 structural controls
   (Histone H3, DNA1, DNA2) → arcsinh residuals → joint PCA compression to 3 components
   → z-score in PCA space.
   Architecture: ResNet50-UNet, in_channels=3, random conv1 init.
   Training: 11 separate models, one per target marker. Jackson2020 → Danenberg2022.

2. Joint ResNet-UNet
   Preprocessing: Same regression residuals + arcsinh, but NO PCA. Each of 10 residual
   channels z-scored independently using training-set statistics.
   Architecture: ResNet50-UNet, in_channels=10, random conv1 init.
   Training: Single model jointly trained on all 11 marker-prediction tasks. Model has no
   explicit marker identity — must infer from channel ordering (which shifts per task,
   making it genuinely marker-agnostic).

3. Joint ResNet-UNet + HyperConv
   Preprocessing: Identical to condition 2.
   Architecture: ResNet50-UNet with MarkerStem replacing conv1. MarkerStem receives
   Jackson vocabulary indices of the 10 input markers, looks up learned embeddings,
   projects each to a 7x7 kernel, applies per-marker convolutions, sums to 64-channel
   output. Parameter count matched to condition 2.
   Training: Same joint multi-task setup; marker_ids passed per batch.
   NOTE: Run in POC_MODE — 10 epochs only, patience=3. Results are undertrained.

========================================
PROPOSED ABSTRACT
========================================

Cross-cohort virtual staining of IMC data — predicting markers missing from one study's
panel using another study's measurements — is a pressing practical problem: dozens of
breast cancer IMC cohorts exist with incompatible panels, and integrating them requires
either re-staining (expensive, often impossible on archival tissue) or computational
imputation. Current state-of-the-art approaches (VirTues, EVA, ImmuVis) address panel
heterogeneity through increasingly large foundation models pretrained on tens of millions
of patches, effectively throwing compute and parameters at the problem. We argue that the
bottleneck is not model capacity but preprocessing: standard arcsinh + filter + min-max
normalization treats each channel as an independent intensity signal, ignoring the fact
that a large fraction of any marker's pixel-level variance is structurally driven —
explained by nuclear density and DNA content rather than true antibody expression. We
propose a biologically-grounded preprocessing pipeline that explicitly removes this
structural component via cross-marker regression on control channels (Histone H3, DNA1,
DNA2) before model training, forcing the network to learn residual marker-specific
expression patterns. Using a compact ResNet-UNet trained from scratch on a single breast
cancer cohort (Jackson2020), we show this preprocessing alone enables competitive
cross-cohort generalisation to an independent breast cancer dataset (Danenberg2022),
matching or outperforming large pretrained foundation models on the same task — without
their computational requirements. This makes the approach accessible to the typical IMC
lab, which has one or two cohorts, a single GPU, and no path to pretraining on 17 million
patches.


========================================
KEY FOLLOW-UP EXPERIMENTS
========================================

1. PREPROCESSING PIPELINE ABLATION (priority)
   Implement ImmuVis-style preprocessing on the same ResNet-UNet architecture and run
   the same Jackson→Danenberg evaluation.
   ImmuVis preprocessing: arcsinh(x/5) + Butterworth low-pass filter (order 2, cutoff
   0.2 Nyquist — see Uchal et al. 2026 Appendix A.7) + panel-wise 99th-percentile
   min-max normalization.
   Compare per-marker and joint ResNet-UNet under:
     (a) Regression-residual preprocessing (ours)
     (b) ImmuVis-style preprocessing
   This directly tests whether the preprocessing is the key ingredient for cross-cohort
   generalisation. VirTues comparison not needed — ImmuVis already demonstrates
   superiority over VirTues under both preprocessing pipelines (their Tables 1 and 8).

2. FID AS STANDARD EVALUATION METRIC
   ImmuVis, VirTues, and EVA report only MSE and Pearson — no spatial plausibility
   metric. Introduce FID as a complementary evaluation axis:
     - Show that high Pearson does not guarantee biologically plausible spatial structure
     - Per-marker FID breakdown: structural/epithelial markers (CK5, panCK) show
       largest FID gaps between conditions
     - Argue FID should be standard in IMC virtual staining benchmarks
     - Optional: compute FID of ImmuVis pretrained checkpoint zero-shot on J→D task
